# 功能说明与产品边界

## 产品定位

One Wiki 是一个 AI 增强型企业知识平台。它为小型团队提供统一的知识入口，目标是让员工能够在短时间内找到、理解和验证需要的专业知识。

平台围绕“知识主题”和“学习/检索任务”组织信息，不围绕部门结构或项目过程组织信息。

## 目标用户

| 角色 | 使用目标 |
| --- | --- |
| Reader | 查找、阅读、收藏和询问知识 |
| Contributor | 创建并维护自己负责的知识草稿 |
| Editor | 编辑、发布和归档知识内容 |
| Admin | 管理用户、主题、系统设置和平台运行状态 |

## 当前功能

### 知识内容

- 创建知识草稿，填写标题、摘要、主题、标签和正文。
- 使用层级化正文内容表达知识；第一版支持轻量 Markdown 标题和段落。
- 草稿与已发布内容隔离，草稿不会出现在读者搜索或 AI 引用中。
- 发布时生成不可变版本，并记录发布说明、创建人和时间。
- 支持归档不再有效的知识。
- 阅读页显示所属主题、负责人、版本、更新时间和标签。

### 发现与检索

- 首页展示主题入口与最近更新的已发布知识。
- 支持标题、摘要、正文的关键词搜索。
- 页面按主题和标签提供上下文，帮助读者从概览进入细节。
- AI 问答优先检索已发布页面，并返回引用页面与摘要。

### AI 问答

- 通过统一的 OpenAI-Compatible 接口接入生成模型。
- 生成模型不可用或未配置时，退化为“可信检索结果”模式，不伪造答案。
- 提示词要求模型只依据检索到的知识回答；证据不足时明确说明。
- 引用包含页面标题、跳转地址和相关片段。

### 文件

- 附件保存到可配置的本地目录，Windows 和 Linux 均可使用。
- 实际文件使用 UUID 路径保存，原始文件名仅作为元数据保存。
- 上传时限制体积、屏蔽可执行文件、计算 SHA-256，并使用临时文件后原子移动。
- 下载经后端认证，上传目录不直接暴露为公开静态目录。

### 权限与安全

- 邮箱/密码登录，密码采用 `scrypt` 加盐哈希。
- 会话以 HttpOnly Cookie 保存，并在数据库中保存令牌摘要和过期时间。
- API 层执行角色校验，不能只依靠前端隐藏按钮。
- 页面读取、编辑、发布、归档、主题创建和附件上传分别受角色约束。

## 内容生命周期

```text
创建草稿 → 编辑和保存 → 发布新版本 → 搜索 / AI 引用
                                       ↓
                                  归档或等待复审
```

## 明确不做

- 部门空间、项目空间和会议管理
- 任务、审批、故障管理和即时通讯
- 实时多人编辑、在线光标和协作白板
- AI 自动发布知识或执行外部操作
- 将 AI 生成内容当作没有来源的事实

## 下一阶段优先级

1. 使用 Tiptap 替换轻量 Markdown 编辑器，支持结构化富文本、表格、图片和块级引用。
2. 将 PostgreSQL + pgvector 作为生产数据库与语义检索层，并加入混合召回与重排。
3. 引入 Redis/Celery，异步处理附件抽取、Embedding、重建索引和复审提醒。
4. 支持 OIDC/LDAP 登录、学习路径、阅读历史、反馈闭环和内容复审工作流。
